# Apply Mate - Complete API & Architecture Guide

## Overview
Your app is a job application tracker with a Node.js/Express backend and React frontend. They communicate via REST APIs using the 4 main HTTP methods: **GET, POST, PUT, DELETE**.

---

## 🔙 BACKEND STRUCTURE

### 1. **Models** (`backend/models/Job.js`)
- **Database Schema**: Defines the structure of a job document in MongoDB
```javascript
{
  _id: ObjectId,              // Auto-generated by MongoDB
  companyName: String,        // Required
  jobRole: String,            // Required
  status: String,             // 'applied', 'interviewing', 'rejected', 'offer'
  date: String,               // Date applied (YYYY-MM-DD)
  jobLink: String,            // Job posting URL
  notes: String,              // User notes
  createdAt: Date,            // Auto-added by timestamps
  updatedAt: Date             // Auto-added by timestamps
}
```

### 2. **Controllers** (`backend/controllers/jobController.js`)
Contains all business logic for handling requests:

#### **GET - Fetch All Jobs**
```javascript
export const getJobs = async (req, res) => {
    try {
        const jobs = await Job.find().sort({createdAt: -1});
        res.status(200).json(jobs);
    }catch(err){
        res.status(500).json({message: err.message})
    }
}
```
- Retrieves all jobs from MongoDB
- Sorts by newest first (`-1`)
- Returns array of jobs

#### **POST - Create New Job**
```javascript
export const createJob = async (req, res) => {
    try {
        const job = await Job.create(req.body);
        res.status(201).json(job);
    }catch(err) {
        res.status(500).json({message: err.message})
    }
}
```
- Takes data from request body
- Saves to MongoDB
- Returns newly created job with `_id`

#### **PUT - Update Job**
```javascript
export const updateJob = async (req, res) => {
    try {
        const updatedJob = await Job.findByIdAndUpdate(
            req.params.id,
            req.body,
            {new: true}
        );
        res.status(200).json(updatedJob);
    }catch (err) {
        res.status(500).json({message: err.message});
    }
}
```
- Uses job ID from URL (`req.params.id`)
- Updates with new data from body
- Returns updated job

#### **DELETE - Remove Job**
```javascript
export const deleteJob = async (req, res) => {
  try {
    await Job.findByIdAndDelete(req.params.id);
    res.status(200).json({ message: "Job deleted successfully" });
  } catch (err) {
    res.status(500).json({ message: err.message });
  }
}
```
- Uses job ID from URL
- Removes from database
- Confirms deletion

### 3. **Routes** (`backend/routes/jobRoutes.js`)
Maps HTTP methods to controller functions:

```javascript
router.post('/', createJob);           // POST /api/jobs
router.get('/', getJobs);              // GET /api/jobs
router.put('/:id', updateJob);         // PUT /api/jobs/:id
router.delete('/:id', deleteJob);      // DELETE /api/jobs/:id
```

### 4. **App & Server** (`backend/app.js` & `server.js`)
- **app.js**: Express setup with middleware (CORS, JSON parsing)
- **server.js**: Starts the server and connects to MongoDB

## 🎨 FRONTEND STRUCTURE

### 1. **Context & State Management**

#### **JobContext** (`src/context/JobContext.jsx`)
- Creates a React Context to share state globally
- Avoids passing props through many levels

#### **JobProvider** (`src/provider/JobProvider.jsx`)
- **Fetches jobs on app startup** with `useEffect`
- **Makes GET request** to `http://localhost:5000/api/jobs`
- **Stores jobs in Redux-like reducer** using `useReducer`
- **Provides dispatch function** to all components via Context

```javascript
// On component mount, fetch all jobs from backend
useEffect(()=> {
    const fetchJobs = async ()=> {
        const res = await fetch('http://localhost:5000/api/jobs');
        const data = await res.json();
        dispatch({ type: "SET_JOBS", payload: data })
    }
    fetchJobs();
}, [])
```

#### **jobReducer** (`src/reducers/jobReducer.js`)
- Manages state updates for all job operations
- Actions: `ADD_JOB`, `DELETE_JOB`, `EDIT_JOB`, `SET_JOBS`
- Uses `_id` (from MongoDB) not `id`

### 2. **Pages**

#### **AddJob.jsx** - CREATE Operation
**Flow**: Form → POST request → Backend saves → Response → Update state

```javascript
async function createJob(e){
    e.preventDefault();
    const res = await fetch('http://localhost:5000/api/jobs', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(formData)
    });
    const newJob = await res.json();  // Get job with _id

```

**Key Points:**
- Status must be lowercase: `'applied'`, `'interviewing'`, etc.
- Backend generates `_id` automatically
- Form resets after successful submission

#### **Dashboard.jsx** - READ & Filter
- Displays statistics (total, applied, interviews, offers, rejected)
- Filters jobs by status
- Shows recent jobs in a list

**Important**: Filter uses lowercase status
```javascript
const appliedLength = jobs.filter(job => job.status === 'applied').length;
```

#### **JobsMenu.jsx** - READ & Display
- Shows list of all jobs using `JobCard` component
- Gets jobs from Context via `useJobs()` hook

### 3. **Components**

#### **JobCard.jsx** - UPDATE & DELETE Operations
Displays individual job with Edit/Delete buttons:

**DELETE:**
```javascript
async function handleDelete(id) {
    const res = await fetch(`http://localhost:5000/api/jobs/${id}`, {
        method: 'DELETE'
    });
    dispatch({ type: "DELETE_JOB", payload: id });
}
```

**EDIT:**
- Opens `ModalEdit` modal with job data

#### **ModalEdit.jsx** - UPDATE Operation
**Flow**: Edit form → PUT request → Backend updates → Response → Update state

```javascript
async function updateJob(e){
    e.preventDefault();
    const res = await fetch(`http://localhost:5000/api/jobs/${editingJob._id}`, {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(editingJob)
    });
    const updatedJob = await res.json();
    dispatch({ type: 'EDIT_JOB', payload: updatedJob });
}
```

## 📋 COMPLETE API FLOW - ALL 4 METHODS

### **Method 1: POST (Create)**
```
Frontend (AddJob.jsx)
    ↓
fetch POST /api/jobs with form data
    ↓
Backend (jobController.createJob)
    ↓
Job.create() saves to MongoDB
    ↓
Returns new job with _id
    ↓
Frontend receives response
    ↓
dispatch ADD_JOB → Update local state
    ↓
UI re-renders with new job
```

### **Method 2: GET (Read)**
```
Frontend (JobProvider.jsx on mount)
    ↓
fetch GET /api/jobs
    ↓
Backend (jobController.getJobs)
    ↓
Job.find() queries MongoDB
    ↓
Returns array of all jobs
    ↓
Frontend receives array
    ↓
dispatch SET_JOBS → Update local state
    ↓
UI renders all jobs
```

### **Method 3: PUT (Update)**
```
Frontend (ModalEdit.jsx)
    ↓
fetch PUT /api/jobs/:id with updated data
    ↓
Backend (jobController.updateJob)
    ↓
Job.findByIdAndUpdate() modifies MongoDB
    ↓
Returns updated job
    ↓
Frontend receives response
    ↓
dispatch EDIT_JOB → Update local state
    ↓
UI re-renders with changes
```

### **Method 4: DELETE (Remove)**
```
Frontend (JobCard.jsx)
    ↓
fetch DELETE /api/jobs/:id
    ↓
Backend (jobController.deleteJob)
    ↓
Job.findByIdAndDelete() removes from MongoDB
    ↓
Returns success message
    ↓
Frontend receives response
    ↓
dispatch DELETE_JOB → Update local state
    ↓
Job disappears from UI
```

## 🚀 IMPORTANT POINTS

### **Status Values** 
- **Backend (database)**: lowercase `'applied'`, `'interviewing'`, `'rejected'`, `'offer'`
- **Frontend display**: Use capitalize function for UI
- **Form submit**: Convert to lowercase before sending

### **ID Handling**
- **Stored as**: `_id` (MongoDB's ObjectId)
- **In reducer**: Use `_id` for filtering/mapping, not `id`
- **In frontend form**: Send `editingJob._id` for updates/deletes

### **CORS Configuration**
- Backend enables CORS with `app.use(cors())`
- Allows frontend (localhost:5173) to communicate with backend (localhost:5000)

### **Error Handling**
- All async operations wrapped in try-catch
- User gets alert on failure
- Loading state prevents multiple submissions
